# Прототип RAG-системы CodeLens: Семантический поиск по коду

Решение команды в рамках Чемпионата ГК «Ростелеком» (2-й этап), направление «Искусственный интеллект / Data Engineering».  
Проект представляет собой интеллектуальный ИИ-поиск по исходному коду FastAPI-приложения `gymhero`.

---

## 🚀 Особенности нашего решения

* **Гибридный поиск (Hybrid Search):** Сочетает векторную семантику (модель `paraphrase-multilingual-MiniLM-L12-v2`) и текстовый скоринг (дополнительные бонусы за точное совпадение ключевых слов в названиях функций и теле кода). Это даёт высокий показатель метрики `Precision@5`.
* **Умное чункование через AST:** Код разбивается не по фиксированному числу символов, а на основе анализа абстрактного синтаксического дерева (библиотека `ast`). Мы выделяем логические блоки: целиком классы и функции вместе с их docstring-документацией.
* **Быстрое хранилище:** Локальная векторная база данных `ChromaDB` обеспечивает мгновенный отклик при поиске.
* **Интеграция с локальной LLM:** В веб-интерфейсе можно включить режим ИИ-ассистента, который анализирует найденные фрагменты кода через локальную модель `mistral:7b` (в связке с Ollama) и выдает краткий технический ответ на русском языке.

---

## 🛠 Архитектурная схема системы

1. **Индексация:** `index.py` → Парсинг папки с кодом через `ast` → Эмбеддинги через `sentence-transformers` → Сохранение в `ChromaDB` (каталог `./chroma_db`).
2. **Веб-интерфейс:** `app.py` на `Streamlit` принимает текстовый запрос от пользователя.
3. **Поиск:** Модуль `search_engine.py` сравнивает вектор запроса с базой и считает косинусное сходство + добавляет keyword-бонусы.
4. **Генерация ответа:** Модуль `llm_assistant.py` отправляет топ-5 фрагментов в Ollama для генерации суммаризации.
